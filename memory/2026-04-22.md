### Learning: Silent No-Op Bug Pattern
**AdamW was silently doing nothing with Matrix objects.** `param[i]` and `param.length` are `undefined` for Matrix objects (which use `.data` typed array). The optimizer loop ran zero iterations and modified nothing. Tests passed because they used plain arrays.

**Pattern to watch for:** Any code that indexes into objects with `obj[i]` or checks `obj.length` may silently fail with typed wrapper objects. Always test with the actual types used in production code.

**Root cause:** Insufficient type coverage in tests. The optimizer tests used `Float64Array` directly but the actual usage passes `Matrix` objects. This is a _boundary bug_ — the interface works for one type but not the type actually used.
