### Key Bug Fix
**Query cache not invalidated on ROLLBACK/COMMIT** — this was the stealthiest bug yet. The transaction engine, MVCC, heap storage, and scan interceptors were all correct. The bug lived in the query cache layer, which is as far from "transaction rollback" as you can get while still being in the query path. Lesson: when debugging multi-layer systems, check caches early.

### Lessons
- **Databases are layer cakes** — a "rollback bug" can be in the cache, not the rollback code
- **Adaptive query engine + caching = hidden state** — bypassing _connExecute means the adaptive engine can pollute the cache with transaction-local results
- **Savepoints are elegant** — the undo log pattern makes them almost free to implement once you have it
