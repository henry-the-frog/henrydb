# Lessons from Session C (2026-04-19 Evening)

## Variable Renames Need Grep Verification
When renaming a variable in a large file (10K+ lines), always `grep` for ALL references.
The pg-server `options`→`connOptions` rename missed one reference at line 1071, which
broke every single query through the wire protocol (71 test files failed).

**Rule:** After any variable rename in a file >500 lines, run `grep -n 'oldName' file`
before committing.

## Non-Unique B-Tree search() is a Footgun
`search(key)` returns the FIRST match. For non-unique indexes, there can be multiple
rows with the same key. This caused UPDATE and DELETE to only affect the first matching
row instead of all matches.

**Rule:** For equality lookups on potentially non-unique indexes, ALWAYS use
`range(key, key)` which returns all matches. Never use `search()` for DML operations.

## Parser Keyword Conflicts
SQL keywords (DEPTH, CYCLE, WINDOW) used as column names get uppercased by the parser.
If the schema stores column names lowercase and the executor compares case-sensitively,
UPDATE SET on these columns fails with "column not found."

**Rule:** Column lookups should always be case-insensitive or normalize to a consistent case.

## Correlated Subquery Decorrelation is Simpler Than Expected
Converting `WHERE x IN (SELECT y FROM t2 WHERE t2.key = t1.key)` doesn't require
rewriting the AST into a semi-join. Instead:
1. Extract correlation predicates
2. Execute inner query once without correlation
3. Build Map<outerKey, Set<innerVal>> hash map
4. Look up per-row in O(1)

This transforms O(n*m) to O(n+m) with minimal implementation complexity.

## || vs CONCAT() Have Different NULL Semantics
- `||` (SQL standard): NULL propagates → `'hello' || NULL` = NULL
- `CONCAT()` function: NULL treated as empty string → `CONCAT('hello', NULL)` = 'hello'

If the parser converts both to the same AST node, you lose this distinction.
**Rule:** Keep operator and function semantics separate in the AST.
