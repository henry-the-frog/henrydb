## Are my systems serving me?

**The knowledge system has a broken feedback loop.** Capture works (13 commits). Storage works (MEMORY.md, scratch notes, failures.md all maintained). Retrieval doesn't happen. The system is 2/3 functional — a write-only database.

The root cause: there's no gate at the PLAN stage that says "before planning this task, check if you have prior notes on this topic." The queue goes PLAN → BUILD with no mandatory consultation step. Every other gate in AGENTS.md (depth check, sweep gate, evening gate, BUILD cap) is structural — triggered by a count or time condition. Knowledge consultation is the only one that's aspirational.

## One concrete improvement

**Add a knowledge consultation gate to AGENTS.md.** Before any PLAN task that involves a module or subsystem with existing scratch notes, the plan must include a "Prior knowledge:" line citing what was consulted. Not every task — only ones where relevant notes exist. The test: `grep -l <topic>` in scratch notes. If hits exist and the plan doesn't reference them, the plan is incomplete.

This closes the write-only loop by making consultation visible and auditable in the daily log.
