# Reflection: 2026-04-28 Session B

## Key Findings

### HeapFile Array→Map (88,000x improvement)
The single most impactful discovery: `HeapFile.pages` was an `Array` using `.find()` for page lookups. Every heap.get/delete/update/scan did a linear search. Switching to `Map` made page access O(1). This is a textbook case of "the obvious data structure matters more than clever algorithms."

**Lesson:** Always check how frequently-called methods resolve their targets. `.find()` on arrays is the silent performance killer.

### V8 JIT Cold-Call Latency
`_evalExpr()` first call: 846μs. After warmup: 0.39μs. 2,169x difference! The "slow UPDATE" was V8 compiling the expression evaluator's large switch-case tree on first invocation. The compiled-expr approach (`new Function()`) pre-compiles, giving fast first-call performance.

**Lesson:** In benchmarks, always warm up. In production code, consider pre-compilation for hot paths. First-call latency is real and measurable.

### Compiled Expression Safety
Compiled WHERE expressions broke correlated subqueries because `table.column` references were compiled as `row["column"]` (inner scope) instead of the outer scope's value.

**Lesson:** When JIT-compiling, scope awareness is critical. A qualified column name means "look outside," not "look at a different key in the same row."

### WeakMap + In-Place Mutation = Bug
Prepared statements use in-place AST mutation (fastBind). WeakMap cached the compiled expression keyed on the AST root. But the root doesn't change — only children do. So the cache returned stale functions.

**Lesson:** Caching by object identity fails when the object is mutated in place. Either don't cache, or use structural identity (hash of content).

### WASM GC in Node.js v22
Struct types compile and execute without flags. struct.new returns opaque GC-managed objects. Inner WASM loops with GC structs are 21x faster than equivalent JS loops. This opens a path to a GC backend for monkey-lang.
