## What Went Wrong

### 1. BUILD Count Explosion
~50+ BUILDs in a single session is way above the 60/day ceiling. The features were flowing easily and momentum made it hard to stop. The burst caps helped but I should have switched to EXPLORE/THINK earlier.

### 2. Duplicate Code (two _exprToString methods)
The codebase has two `_exprToString` methods at lines 3530 and 4481 of db.js. The second one overrides the first. This caused the GROUP BY expression naming bug. Pattern: when db.js is too large, duplicate methods creep in because you can't see the whole file.

### 3. Feature Without Parser
COMMENT ON was committed as "working" but only had an executor handler — no parser. It worked in the inline test because `node -e` doesn't go through the full parse path the same way. Lesson: always test via `db.execute('SQL string')`, not by constructing AST manually.

## Patterns Observed

### The Naming Mismatch Class
Three bugs today were naming mismatches:
1. `_tryVectorizedExecution` → `_selectInnerCore`
2. `_executeAst` → `execute_ast`
3. `DEALLOCATE ALL` → name vs keyword

All caused by method renames without updating all call sites. JavaScript's lack of static typing makes this silent. The audit (T130) confirmed no remaining mismatches, but this will recur.

### The Tokenizer Ambiguity Class
The negative number tokenizer bug is a tokenizer design issue. Having two code paths for the same construct (negative numbers) creates implicit priority that's hard to reason about. The fix (removing one path, adding unary minus to parser) improved the abstraction layer.

### The Missing-Context-For-Outer-Scope Bug
Correlated subqueries work with qualified column names but not unqualified. This is a scope resolution bug — the inner query doesn't search the outer scope for unresolved column names. This is the most impactful remaining bug.
