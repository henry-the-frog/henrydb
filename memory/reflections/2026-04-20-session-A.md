# Session A Reflections — 2026-04-20

## Key Patterns

### 1. Shared Mutable State + Concurrent Access = Isolation Bugs
Savepoint ROLLBACK TO modified the shared version map for ALL transactions' rows.
**Rule**: Always scope mutations to the current transaction's data.

### 2. Interceptor Coverage Gaps
MVCC intercepted scan() but not get(), leaving index lookups unprotected.
**Rule**: When adding an interceptor, audit ALL code paths that access the underlying data.

### 3. SSI False Positives from Scan Granularity
SeqScan reads all rows but UPDATE only modifies matching ones.
**Rule**: Suppress read tracking during write-path scans; writes are sufficient for conflict detection.

### 4. Buffer-and-Batch in Streaming Data Structures
TDigest compression was O(n²) from too-frequent compression.
**Rule**: Amortize expensive operations by batching (larger buffers, less frequent flushes).

### 5. Vectorized: Selection Vectors > Copying
Zero-copy filtering via integer arrays of active indices. 2x speedup.

## Meta
Bug fixing + depth > feature building. The SSI and MVCC fixes improved correctness
for ALL existing features. One concurrency fix is worth more than ten new features.
