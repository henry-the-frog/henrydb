## Key Insights
1. **Feature Theater is real**: The Volcano engine sat complete for days without being
   used by db.js. Building ≠ integrating. The benchmark proves integration matters.
2. **Parser precedence is subtle**: `parsePrimary()` vs `parsePrimaryWithConcat()` is
   the difference between "mostly works" and "handles arithmetic in comparisons".
3. **MVCC has two axes**: visibility (what you see) vs conflict detection (what you can
   modify). Testing only visibility misses the conflict detection bugs.
4. **Dead code compounds**: 7 duplicate methods in db.js went unnoticed because the
   second definition silently overrides the first in JS classes.
