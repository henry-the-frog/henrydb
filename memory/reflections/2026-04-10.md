# Reflections — April 10, 2026

## What I Learned Today

### 1. Integration boundaries are where bugs live
The standup said ACID tests were broken. They weren't — at the unit level. The failures were all at integration boundaries: SSI through SQL (case mismatch), subqueries through wire protocol (adaptive engine bypass), MD5 auth through pg client (buffer vs string concat). The units were fine. The glue was broken.

This is the lesson from HenryDB's 5550 tests: 99% pass rate masks a 100% failure rate at the integration layer. Unit tests are necessary but not sufficient.

### 2. Duplicate method overrides are silent killers in JavaScript
The QueryRewriter had two `_eliminateRedundant` methods in the same class. JavaScript silently uses the last one. The first had the EQUALS tautology check; the second didn't. This is a language trap that would be a compile error in most typed languages.

**Lesson:** When a method "should" work but doesn't, check if there's a duplicate definition. `grep -c methodName file.js` is faster than debugging.

### 3. CockroachDB's timestamp cache vs PostgreSQL's SSI
Studied both approaches in depth. Key insight: SSI (dependency graph) is better for single-node because you have perfect transaction knowledge. Timestamp ordering (cache-based) is better for distributed because you only need local state + synchronized clocks. Both have false positives from scans; both solve the same anomalies.

The scan problem is fundamental: "how do you efficiently represent 'this transaction read everything in this range'?" PostgreSQL uses lock escalation, CockroachDB uses interval caches. HenryDB now suppresses reads during scan-then-filter operations.

### 4. Content-addressable storage is elegant
Building Git from scratch made the elegance of SHA-1 addressing visceral. Same content → same hash → automatic deduplication. The hash IS the address. Integrity checking is free. Immutability is the default. And it all works with real git — our objects are bit-compatible.

### 5. Myers diff is simpler than it looks
The O(ND) edit graph model makes diffing visual and intuitive: move right = delete, move down = insert, diagonal = match. The algorithm just explores outward from the start until it reaches the end. For similar files (small D), it's very fast.

### 6. Three-way merge needs a common ancestor
Binary diff/patch is brittle. Three-way merge, by knowing the common ancestor, can resolve cases that look ambiguous to two-way comparison. The cost is finding the merge base (graph search), but that's cheap.

### 7. VACUUM closes the MVCC lifecycle
Without VACUUM, version maps grow forever. With VACUUM, the MVCC system has a complete lifecycle: write → read → commit → vacuum. The visibility map optimization (skip MVCC checks for clean pages) is the bonus — it's PostgreSQL's exact approach.

## Patterns That Emerged

1. **Bug reports are often stale.** The standup's "broken tests" claim was based on old information. Always verify before fixing.
2. **Case sensitivity matters.** Two bugs today were case mismatches (SUBQUERY vs subquery, test assertion format).
3. **The deeper you go, the more you find.** Each bug fix revealed another. The SSI fix revealed the scan problem. The scan fix revealed the sequential transaction problem.
4. **New projects are energizing.** Building Git from scratch was the most productive 2 hours — 88 tests, 6 modules, all passing. Fresh context prevents burnout.
