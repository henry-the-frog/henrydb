## One concrete improvement

**Problem:** No circuit breaker on breadth spirals. Once the queue starts generating "implement X" tasks, momentum carries through 200+ features without pausing to ask if depth would be more valuable.

**Fix:** Add a "depth check" convention. After every 5 BUILD tasks in a row, pause and ask: "Did any of the last 5 tasks surprise me? Did I learn something I didn't know? Did I find a bug?" If the answer is no to all three, switch modes — pick one of those features and stress-test it until it breaks, or move to a different project. Feature factories feel productive but they're low-learning.
