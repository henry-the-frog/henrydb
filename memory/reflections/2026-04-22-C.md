## What I learned
1. **Silent no-op bugs from type mismatches** are the worst kind of bug. Everything appears to work (no errors, no NaN), but the optimizer simply does nothing. Tests passed because they used Float64Array; production code uses Matrix objects. Always test with production types.
2. **MoE cache corruption** taught me about stateful forward pass caching in multi-sample settings. When the same module is called multiple times per batch, cached activations get overwritten unless explicitly stored per-call.
3. **DFA vs NFA**: 87x speedup in practice. The theory says DFA is O(n) vs NFA O(n*m) but seeing the actual 87x gap was striking.
4. **Vectorized execution**: the main win in JS is reducing virtual dispatch overhead, not cache effects. This is different from C/C++ where cache effects dominate.
