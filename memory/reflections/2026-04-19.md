## Key learnings
1. **Variable renames in large files need grep verification** — the `options`→`connOptions` rename missed one reference and broke ALL wire protocol queries. Simple typo, massive blast radius.
2. **Non-unique index search() vs range()** — `search()` returns one match, `range(val,val)` returns all. This is a common footgun in B-tree implementations.
3. **Correlated subquery decorrelation doesn't need full AST rewriting** — batch the inner query, build a hash map, done. Much simpler than converting to a semi-join AST node.
4. **Parser keyword conflicts** — when SQL keywords are used as column names, case mismatches cause subtle bugs in the executor.
