## Did I actually learn today?

**Yes, and the learning was concentrated in the depth sessions.**

High-value insights:
- JS `&&/||` !== SQL three-valued logic (`null || false` → `false` in JS, should be `null` in SQL) — systemic, affects multiple operators
- Try-catch + property-order: getter-only property assignment fails silently inside try block, aborting the entire recovery sequence
- Undo-log design: naive reverse-order fails for chained ops on same row (INSERT→UPDATE→DELETE = stale RIDs). Snapshot approach is simpler AND correct
- HOT chain heads must preserve redirects even after physical data pruning
- Predicate pushdown through views/CTEs requires re-qualifying column references against the view's output schema

The morning feature work produced lower learning density — TCO and UDFs were implementations of known patterns. The evening bug hunting produced insights that transfer to any database implementation.
