**Things learned:**
- PG wire protocol type OIDs: The pg client library determines JS types from the OID in RowDescription. Missing OIDs default all values to TEXT strings. The type map needs to be comprehensive.
- BackendKeyData: PG clients expect 'K' message after auth for process ID tracking. Missing it → processID=0.
- Snapshot visibility: The correct write-write conflict check uses xmax + activeSet, not xmin. xmin catches too many committed transactions.
- EXPLAIN cost model: For equality lookups on indexed columns with selectivity ≤ 0.5, prefer index scan even if seq scan is marginally cheaper.
