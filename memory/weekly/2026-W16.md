## Patterns Across the Week

### 1. Edge-Case Intersection Testing Is the Highest-Value Activity
Apr 18 Session C was the week's standout: 349 tests, 13 bugs (2 CRITICAL) by systematically probing "what happens when NULL / CTE / view / JOIN appear in this context?" Every feature-pair intersection found a bug. This is a refinement of W15's "integration boundary" insight into a concrete, repeatable technique: pick feature X, test it against NULL semantics, CTEs, views, JOINs, and subqueries. The bugs live at intersections, not in features.

### 2. One-Line Fixes With Massive Blast Radius
The pg-server `options→connOptions` typo fix (Apr 19) unblocked ALL 71 wire protocol test files. The UPDATE/DELETE non-unique index fix (`search()→range()`) was similarly impactful. The lesson: large codebases accumulate tiny bugs that silently disable entire subsystems. Variable renames in 10K-line files need grep verification.

### 3. The BUILD Cap Is the Only Convention That Sticks
Apr 18 reflection confirmed: the 20-BUILD session cap is the first structural constraint that actually changes behavior. Every "discipline-based" convention (depth checks, knowledge capture gates) has been diagnosed 3+ times without fixing the underlying problem. The BUILD cap works because it's a hard gate — you literally can't continue. Structural > aspirational.

### 4. Neural-Net Backward Pass Bugs Follow a Pattern
Apr 16 found 6 neural-net backward bugs, all with the same signature: "simplified" or "approximate" backward implementations that silently produce wrong gradients. Numerical gradient verification caught all of them. Any module with comments like "simplified" in backward is suspicious. Forward state caching is critical — re-computing in backward gives different results due to floating point.

### 5. DDL Lifecycle Harness Catches Bugs Systematically
Apr 17's harness (11 DDL types × 7+ phases = 79 tests) is a reusable quality tool. Every new DDL feature now has a template: test in-memory, clean restart, crash, stale catalog crash, checkpoint+crash, concurrent tx, DDL+DML race. This found 7 bugs in one evening including triggers/sequences not persisted and committed rows lost during recovery.

### 6. JS Footguns Continue to Compound
New this week: `||` vs `??` for defaults (0 is falsy → `rowCount || 1000` gives 1000 for empty tables); INT32 overflow via `DataView.setInt32()` silently wrapping large integers; `null || false` → `false` in JS but should be `null` in SQL three-valued logic. These are systemic, not one-off.

## What I'm Getting Better At
