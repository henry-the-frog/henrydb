**Bugs found:** (all with root causes)
1. Window ORDER BY alias: __window_ prefix not checked during sort
2. ROWS BETWEEN parser: missing frame clause keywords
3. CTE UNION ALL: split query representation (query + unionQuery) only handled in recursive path
4. View UNION resolution: _select() doesn't handle UNION type, needs execute_ast()
5. ROLLBACK no-op: BEGIN/ROLLBACK were just flag toggles, no snapshot/restore
6. CTE column aliasing: Object.keys() loses duplicates, need AST-based positional mapping
7. HAVING complex expressions: _serializeExpr() missing for aggregate key generation
8. Derived table columns: alias prefix (sub.col) not stripped during resolution
9. Join ordering ON-dependency: DP set connectivity ≠ sequence dependency, need greedy emission
10. ORDER BY alias: aliasExprs only captured expression/function types, not column with alias
11. Hash join NULL=NULL: String(null)="null" caused NULL key matching

**Things learned:** (genuine insights)
- Feature flags vs implementation: ROLLBACK *looked* transactional without *being* transactional
- System R DP: set-level optimization ≠ sequence-level dependency ordering
- Expression serialization: needed for consistent aggregate key matching in HAVING
- CTE dual representation: query + unionQuery requires parallel handling in all code paths
- aliasExprs must capture ALL column types with aliases
- Hash join: String(null) = "null" is a subtle type coercion bug
