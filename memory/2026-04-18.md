### Bugs Found (4!)
1. **CRITICAL: Heap page overflow** — insertTuple returns -1 but rowCount incremented anyway. Row stored with corrupted RID (-1 slotIdx), unreadable via scan(). Silent data loss.
2. **UNIQUE constraint + HOT chains** — Index-based constraint check didn't verify rows existed in heap. Stale entries from deleted HOT-updated rows caused false UNIQUE violations.
3. **Extended query column naming** — getColumns used raw SQL fragments ("LENGTH(val) as len") as column names. pg client received wrong field names.
4. **Describe executing queries** — Portal Describe handler was calling db.execute() on the actual bound SQL, causing side effects.

### Things Learned
- PG wire protocol extended query flow: Parse→Bind→Describe(Portal)→Execute→Sync
- Describe MUST return correct RowDescription even for empty dry-run results
- In-memory DB cost model: random_page_cost ≈ seq_page_cost (1.1 vs 1.0)
- Heap page slot corruption: insertTuple return value MUST be checked
- pg npm client uses type OIDs from RowDescription to parse values (numbers vs strings)
